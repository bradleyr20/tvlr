Expected URL Patterns

These are the expected patterns. Note that one or two are still under discussion. See linked xlsx or PDF for color-coded version.

Content type URL model Examples Notes
Slideshows (Galleries) /galleries/YYYY-MM-DD/slug /galleries/2013-12-17/behind-the-scenes-at-the-best-restaurant-in-the-world
Stories /stories/YYYY-MM-DD/slug /stories/2014-01-01/9-reasons-to-visit-colorado-that-arent-pot-tourism All punctuation, apostrophes, etc should be eliminated during translation from title to URL (rather than transformed into dashes; only spaces should become dashes)

/awards/gold-list/2014

Priority of display order:
Stories with both <channel-name> and <subchannel-name> in PRIMARY (ie, first-in-list) positions
Stories with <subchannel-name> in PRIMARY position, <channel-name> in non-primary position
Stories with <channel-name> in PRIMARY posiiton, <subchannel-name> in non-primary position
Stories with NEITHER <channel-name> nor <subchannel-name> in primary position

Venues /<venue-type>/<city>/<name-slug>

/<venue-type>/<city>/<name-slug>-N (where N is an incremental integer starting with 2 and counting upward)		 /hotels/new-york-city/standard-high-line

/hotels/new-york-city/ritz-carlton

/hotels/new-york-city/ritz-carlton-2

/hotels/new-york-city/ritz-carlton-3		 <City> will be required for publication of venues, along with <venue-type>. If there are venues in the same city of the same type and with the same name, deliver error upon attempted publish. Error should advise user that slug is taken, and advise them to either (a) make the venue name more unique (eg, W Hotel Union Square instead of W Hotel), or (b) append a "-<integer>" to the end of the slug.
Ships /<venue-type>/<cruiseLine>/<name-slug> /ships/princess-cruises/the-princess-dawn <cruiseLine> will be required for publication of ships, along with <venue-type>.
